This guide walks you through installing Filament Short URL into an existing Laravel 11\+ application that already has a Filament v5 panel configured. By the end you'll have the plugin running and your first short link ready to share.

## 1. Install via Composer

```bash
composer require janczakb/filament-short-url
```

## 2. Run migrations

```bash
php artisan migrate
```

The plugin ships cross-database migrations (MySQL, PostgreSQL, SQLite) for `short_urls`, `short_url_visits`, `short_url_daily_stats`, `short_url_custom_domains`, `short_url_settings`, plus the v5.2 audit migrations (`external_id`, UTM fields, `domain_scope_id`, cross-dimensional stats, security counts).

## 3. Compile and publish the plugin stylesheet

The plugin uses **Tailwind CSS v4**. After installation (and again whenever you upgrade), compile the plugin's stylesheet from your **host application root**:

```bash
npx @tailwindcss/cli \
  -i ./vendor/janczakb/filament-short-url/resources/css/plugin.css \
  -o ./vendor/janczakb/filament-short-url/resources/dist/filament-short-url.css \
  --minify
```

Then publish the compiled assets so the browser can load them:

```bash
php artisan filament:assets
```

<Warning>
  Skipping this step leaves the Filament panel without plugin styles (QR designer, stats widgets, link forms will render unstyled). Re-run both commands after every `composer update` of this package.
</Warning>

## 4. Register the plugin

Add `FilamentShortUrlPlugin::make()` to your Panel Provider's `plugins()` array. See [Plugin Setup](/configuration/plugin-setup) for all fluent options.

## 5. Schedule the Laravel cron

The plugin auto-registers its scheduled commands (`short-url:aggregate-and-prune` nightly, `short-url:sync-counters` every minute when counter buffering is enabled, `short-url:verify-custom-domains` daily). You only need the standard Laravel scheduler entry:

```bash
* * * * * cd /path-to-your-project && php artisan schedule:run >> /dev/null 2>&1
```

<Note>
  The default queue connection is **`sync`** — the plugin works without a queue worker out of the box. For production set `SHORT_URL_QUEUE=redis` (or `database`) and run `php artisan queue:work`.
</Note>
