<AccordionGroup>
  <Accordion title="v5.2.3 — Production Redis Mode, Stats Scaling, and Settings Diagnostics">
    - **Redis connection in Settings** — When **Queue Connection = `redis`**, configure host, port, password, database, and key prefix in **Settings → General**; values override `database.redis` and `queue.connections.redis` at runtime independently of `CACHE_STORE`.
    - **Test Redis connection** — PING \+ pipelined `INCR`/`SADD` probe directly from the Settings form using unsaved preview values.
    - **Test queue worker** — Dispatches a `VerifyQueueWorkerJob` and waits up to 12 s; reports the exact `php artisan queue:work {connection} --queue={name}` command when no worker responds.
    - **`StatsScalingProfile`** — Auto-enables counter buffering, dedicated Redis counters, and a Redis "today" buffer when queue is `redis`; SQL micro-cache for sync mode.
    - **`VisitCounterBuffer` and `TodayStatsBuffer`** — Pipelined Redis counters for buffered totals/uniques and today's summary; live-merged in the stats cache without busting historical cache per click.
    - **`StatsVisitRecorder`** — Hot-path stats side effects extracted from `TrackShortUrlVisitJob`; per-visit full cache flush removed.
    - **Live feed** — `LiveFeedBroadcaster` prefers the plugin queue Redis over `CACHE_STORE` Redis.
    - **Service provider** — All package migrations registered in `hasMigrations()` including `2026_06_10_000001` and `2026_06_11_000001`.
  </Accordion>

  <Accordion title="v5.2.2 — Enterprise Hardening, Airtight Counters, and i18n Audit">
    - **XSS** — `@json()` used in pixel interstitial and Alpine stats widgets (replaces `addslashes()`).
    - **Atomic `max_visits`** — `VisitSlotReservation::reserveAtomicSlot()` with fast-path SQL and pessimistic lock near cap.
    - **Single-use bots** — BotDetector gate before consuming single-use links.
    - **Buffer double-count fix** — `revertBufferedIncrements()` before DB fallback in `HasVisitCounters`.
    - **API tenancy** — `owner_user_id` required on API keys when `scope_links_to_user` is enabled.
    - **IDOR** — `ResourceOwnershipValidator` for folder/tag/pixel on API writes.
    - **Redirect pipeline** — Limits enforced before app interstitial; caps respected when `track_visits=false`.
    - **`StatsCacheHelper`** — `Cache::lock` on stats miss to prevent stampedes.
    - **`preloadBufferedCountersForIds()`** — Batched buffer reads in list tables.
    - **Public stats** — Uniform 404 and a public-safe response subset.
    - **`HasStats` facade** — Split into `HasVisitCounters`, `HasStatsCache`, `HasStatsQueries`, `HasSecurityStats`.
    - **i18n** — API, redirect, and visitor-facing strings switched to translation keys (EN \+ PL).
  </Accordion>

  <Accordion title="v5.2.1 — Authorization Policy, API Stats Parity, SSE Live Feed, and Load Baselines">
    - **`ShortUrlPolicy`** — `view`, `update`, `delete`, `restore`, `forceDelete`, and `manageSettings` aligned with `scope_links_to_user` / `user_id`; auto-registered when the host has no custom policy.
    - **API filtered stats** — `GET /links/{idOrKey}/stats` accepts the same cross-dimensional filters as the Filament dashboard via `ApiStatsFilterParser` (aliases: `country` → `country_code`, `device` → `device_type`); response includes `meta.date_from`, `meta.date_to`, `meta.filters`.
    - **SSE live feed** — Authenticated `GET /short-url/live-feed/{shortUrl}/stream` replaces Livewire polling; configurable interval and max duration.
    - **Stress baselines** — `php artisan short-url:stress-redirect {key}` benchmark and a `scripts/k6/redirect-baseline.js` k6 load script with p95 thresholds.
  </Accordion>

  <Accordion title="v5.2.0 — SEO & Social, Open Graph, Link Cloaking, and Integration Hardening">
    - **SEO & Social tab** — Per-link `og_title`, `og_description`, `og_image`, `do_index`, `is_cloaked` with a live social preview sidebar.
    - **Open Graph meta** — Per-link OG/Twitter tags for Facebook, LinkedIn, X, Slack, WhatsApp, and other preview bots; SSRF-safe streaming scraper (`GET /short-url/scrape-meta`); WebP image pipeline with temp → permanent promotion.
    - **Search indexing toggle** — `do_index=false` emits `noindex, nofollow` on all responses for that link.
    - **Link cloaking** — `is_cloaked=true` iframe-embeds the destination on your short-link domain for humans while serving a standalone OG page to crawlers; pre-check iframeability via `POST /short-url/check-iframeable`.
    - **REST API expansion** — Bulk create (`POST /links/bulk`, max 100), upsert (`PUT /links/upsert`), `GET /links/exists`, `/random`, `/info`, CSV visit export, tags/folders CRUD, link-level UTM (`utm_source/medium/campaign/term/content`, `ref`), optional `owner_user_id` per API key.
    - **Public stats** — `GET /short-url/public-stats/{url_key}` with optional password (accepted via POST body or `Authorization` header, never query string); rate limited 60/min.
    - **Custom domains** — `domain_scope_id` \+ composite unique `(url_key, domain_scope_id)` allow the same slug on default and each custom domain; multi A-record \+ multi-CNAME verification; activation DNS gate (`SHORT_URL_CUSTOM_DOMAIN_ENFORCE_DNS=true` by default); legacy `/s/{key}` on custom domains 301 → `/{key}`.
    - **Stateless redirect** — Main `/s/{key}` route uses `throttle:120,1` only (no `web` middleware); password flow stays on `/s-auth/{key}` with sessions.
    - **Security hardening** — DNS-aware SSRF guard on outbound URLs (webhooks, OG scraper, iframe checker); webhook re-validated at dispatch with `allow_redirects=false`; Safe Browsing fail-closed \+ cached re-check at redirect; passwords hashed at rest with legacy auto-rehash; bot hardening (Googlebot IP verify, `?bot=1` secret, click dedup); API keys hashed-only; webhook signing required by default.
    - **Filament tenancy** — `LockedUrlKeyGuard` blocks `url_key`/`custom_domain_id` changes server-side; `LinkUserScope` filters resources by `user_id`; API `CustomDomainValidator` enforces domain ownership.
    - **GA4 Measurement Protocol** — Extracted `Ga4MeasurementProtocolService` singleton with full MP payload (`session_id`, `timestamp_micros`, `engagement_time_msec`, standard `page_view` \+ `click`, custom `short_url_visit`); Measurement ID regex validation; real-credential Test connection in Settings.
    - **Geo-IP foolproofing** — `geo_ip_driver=headers` auto-forces `trust_cdn_headers=true`; offline fallback chain Headers → MaxMind → ip-api with a 40-req/min rate limit on the API fallback.
    - **Stats engine** — `FilteredStatsCollector` replaces 15\+ raw GROUP BY queries with daily JSON rollups; `CrossDimensionalStatsEngine` stores single- and two-filter slices in `short_url_daily_stats`; `StatsCacheHelper` driver-agnostic remember/forget; `getSecurityBreakdownStats()` daily bot/proxy rollups; visit logs cursor pagination on `(visited_at, id)`.
    - **Migrations** — `2026_06_08_000001` (`external_id`, UTM, public stats, visits index), `2026_06_09_000001` (`domain_scope_id`, composite unique, FK ON DELETE SET NULL, JSON daily stats dimensions), `2026_06_10_000001` (security counts), `2026_06_11_000001` (cross-dimensional stats); all cross-DB safe.
    - **Default queue** — Remains **`sync`** so the plugin works without a worker; opt-in `database` / `redis` via Settings or `SHORT_URL_QUEUE`.
  </Accordion>

  <Accordion title="v5.0.0 — Live Feed, Folders, Tags, and API Scopes">
    - **Live Activity Feed** is now a full standalone page (`/{record}/stats/live`) with consistent tab navigation alongside Statistics and Visit Logs.
    - The feed uses an O(1) `MAX(id)` poll — when nothing changes, Livewire returns a ~100-byte response, eliminating unnecessary rendering and DOM updates.
    - A 3-second thundering-herd cache collapses concurrent admin users watching the same URL into one database query per window.
    - Country flags in the live feed render via `flagcdn.com` images instead of emoji, with Alpine.js error handling for broken images.
    - **Folders** — Assign each short URL to a folder; clicking a folder in the sidebar filters the link list and shows the link count per folder.
    - **Tags** — Attach up to 5 tags per link; clicking a tag filters the link list to that tag.
    - **Archiving** — Soft-archive links instead of deleting them; archived links are hidden by default and fully restorable.
    - **API key scopes** — Each API key now supports `links:read-only` (GET only) or `links:read-write` (full CRUD); write attempts with a read-only key return `403 Forbidden`.
    - **Per-key rate limiting** — Set an individual request-per-minute cap on each API key independently of the global route throttle.
  </Accordion>

  <Accordion title="v4.0.0 — Custom Domain Branding">
    - **Custom Domains** — Connect branded domains (e.g. `go.company.com`) with real-time CNAME/A record DNS verification and automatic root-level routing without a `/s/` prefix.
    - DNS verification uses PHP's native `dns_get_record()` with a `dig` CLI fallback.
    - Fixed a critical caching bug where single-use links could be visited multiple times during the cache TTL window.
    - Daily aggregation SQL queries now filter only active URL IDs, leveraging composite indexes for better performance.
    - Logo upload API is secured with resource-level permission and policy checks.
    - Settings overrides are deferred to the service provider boot phase to prevent database errors during offline config caching.
    - Added a configurable toggle to disable the global fallback redirection route registration.
  </Accordion>

  <Accordion title="v3.5.0 — User Attribution, A/B Testing, and GDPR IP Anonymization">
    - **User Attribution** — Short URLs now record their creator; avatar and hover card (name \+ email) are displayed in the link list table.
    - **Relative Date Badges** — Creation dates display compact relative strings (`2h`, `5d`, `3mo`) with the precise date visible on hover.
    - **Keyboard Shortcuts** — Press `E`, `Q`, `I`, `S`, or `X` while hovering a table row to edit, view QR code, copy, view stats, or delete.
    - Clicking a table row now navigates directly to the link's Statistics page.
    - **A/B Split Testing** — Distribute traffic across 2–5 weighted variants at root level or nested inside targeting rules, with drag-based sliders and one-click weight balancing.
    - Variant clicks are tracked and displayed as a distribution chart in the analytics dashboard.
    - **GDPR IP Anonymization** — Opt-in IPv4/IPv6 masking with SHA-256 hashing for unique visitor identification, managed via Settings.
    - Google Safe Browsing checks now apply to all URLs submitted via the REST API (destination, A/B variants, and targeting rule URLs).
    - Settings tab query parameters are now human-readable (e.g. `?tab=qr`).
    - Added QR Scan Conversion Rate card with period-over-period delta to the analytics dashboard.
    - The settings form is split into 8 independent single-responsibility tab classes for faster load times and cleaner tab navigation.
  </Accordion>

  <Accordion title="v3.3.0 — Advanced Multi-Filter Targeting Engine">
    - Replaced the legacy single-strategy targeting panel with a flexible rule engine supporting `AND` / `OR` logic matching per rule.
    - New filter categories: **Device** (desktop, mobile, tablet), **Platform** (Windows, macOS, Linux, iOS, Android, Fire OS), **Country** (multi-select with flags), and **Language** (multi-select browser language codes).
    - Country flag icons are dynamically loaded from `flagcdn.com` in both the form and the analytics widgets.
    - Fast-path `getDeviceType` and `getOs` helpers bypass full user-agent regex scanning during redirections for better performance.
    - Legacy single-strategy database records are automatically upgraded to the new multi-filter format when loaded in the Filament form.
  </Accordion>

  <Accordion title="v3.2.0 — Expanded REST API and Alpine.js Webhook Preview">
    - New REST API endpoints: `GET /api/short-url/links/{idOrKey}` (single link details), `GET /api/short-url/links/{idOrKey}/stats` (click metrics), and `PUT/PATCH /api/short-url/links/{idOrKey}` (full update).
    - All API endpoints support lookup by either database `id` or `url_key`.
    - Global REST API route throttling set to 60 requests per minute.
    - Unified validation for API and admin panel forms, including `track_visits`, `ga_tracking_id`, and `auto_open_app_mobile`.
    - Replaced the third-party CodeEditor Filament component with a custom Alpine.js dark-mode webhook payload preview with copy-to-clipboard.
  </Accordion>

  <Accordion title="v3.1.0 — Database-Backed Settings and HMAC Webhooks">
    - Settings are now stored in the `short_url_settings` database table with automatic caching, replacing the legacy JSON file.
    - Legacy `filament-short-url-settings.json` data is auto-imported on first load.
    - Statistics aggregation command rewritten to use database-level `GROUP BY` queries, reducing memory usage to near zero.
    - API keys are now stored as SHA-256 hashes, verified with `hash_equals()`, and shown only once on generation.
    - **HMAC-signed webhooks** — Webhook payloads are now optionally signed via `X-ShortUrl-Signature` using a configurable secret.
    - GA4 Client IDs are now deterministic (hashed IP \+ User Agent) instead of random UUIDs.
    - Single-use, max-visit, and expiring links now force `302` redirects to prevent browser caching from bypassing visit tracking.
  </Accordion>

  <Accordion title="v3.0.0 — Native App Linking, Pixel Registry, and Custom Expiry Pages">
    - **Native App Linking** — Automatically redirect mobile visitors into 24\+ native apps (YouTube, TikTok, Instagram, Spotify, WhatsApp, etc.) using custom URL schemes, with a glassmorphic interstitial page.
    - Live interactive app-matching preview widget inside the Filament edit form.
    - **Global Deep Linking Files** — Serve iOS `apple-app-site-association` and Android `assetlinks.json` from your root domain (disabled by default, configured via Settings).
    - **Central Retargeting Pixel Registry** — Define Meta Pixel, Google Tag, LinkedIn Insight, TikTok Pixel, and Pinterest Tag once, then attach them to any link via checkbox selection in the UI or the REST API.
    - Settings moved from a resource sub-action to a standalone sidebar navigation page.
    - Enhanced browser language targeting with double-pass matching (exact locale first, then base language code).
    - **Custom Branded Expiry Pages** — Expired or inactive links now show a styled page with your site name instead of a bare `410 Gone` error.
  </Accordion>

  <Accordion title="v2.0.0 — QR Logo Upload and Browser Language Tracking">
    - **QR Code Logo Upload** — Upload a custom brand logo inside the QR designer canvas, with configurable sizing, margins, and square/circle shape options.
    - **QR Scan Tracking** — QR code scans are now differentiated from regular web clicks via `?source=qr`, tracked in a dedicated `is_qr_scan` column, and displayed as a distinct metric in analytics and the link list table.
    - **Browser Language Detection** — Visitor preferred language is captured from `Accept-Language` headers and shown in a new **Top Languages** analytics widget.
    - Atomic database transactions for buffered counter updates with fail-safe rollback restoring cache values on DB failure.
    - `getShortUrl()` now correctly handles empty route prefixes, generating `domain.com/abc123` instead of `domain.com//abc123`.
  </Accordion>

  <Accordion title="v1.7.0 — Role-Based Settings Access Control">
    - New `authorizeSettingsUsing(Closure)` plugin method restricts who can access the Settings page using any callable that returns a `bool`.
    - Auto-detects a `manageSettings` method on a registered `ShortUrl` policy as an alternative to the plugin callback.
    - The **Settings** button in the table header is automatically hidden when the current user lacks access.
  </Accordion>

  <Accordion title="v1.6.0 — Google Safe Browsing, VPN Filtering, and World Map">
    - **Google Safe Browsing** — All destination URLs are automatically checked against Google's threat database when a link is created or edited. The Filament UI displays a status badge and blocks saving if the URL is flagged as malware, phishing, or social engineering.
    - **VPN, Proxy, and Tor Filtering** — Detect and optionally block VPN, anonymous proxy, and Tor exit-node traffic to keep your analytics clean. Choose between flag-only mode (logs the visit but marks it) or hard-block mode (returns `403 Forbidden`). Supports IP-API and VPNAPI.io drivers.
    - **Visitor World Map** — A new interactive SVG world map widget on the analytics dashboard visualizes click distribution by country, with per-country hover stats and intensity highlighting.
  </Accordion>

  <Accordion title="v1.5.0 — REST API, Webhooks, and Retargeting Pixels">
    - **Developer REST API** — `GET`, `POST`, and `DELETE` endpoints for managing short URLs programmatically, with API key authentication managed via the Settings UI.
    - **Webhook System** — Real-time HTTP POST notifications on `visited`, `created`, `expired`, and `limit_reached` events, dispatched asynchronously with 3-attempt retry and 10-second backoff.
    - **Social Retargeting Pixels** — Attach Meta Pixel, Google Ads Tag, and LinkedIn Insight Tag to any link; a glassmorphic interstitial fires pixel scripts in the visitor's browser before the redirect.
    - New **Settings → API & Webhooks** tab for managing global webhook URL, monitored events, and API key lifecycle.
  </Accordion>

  <Accordion title="v1.4.0 — Link Expiration, Visit Caps, and Fluent Builder Additions">
    - **Validity Date Ranges** — Set `activated_at` and `expires_at` timestamps to control exactly when a link is active.
    - **Visit Caps** — Define a `max_visits` limit; the link deactivates automatically once the cap is reached.
    - **Custom Expiration Fallbacks** — Redirect expired or over-limit visitors to a custom `expiration_redirect_url` instead of a `410` error.
    - Fluent builder gains `activatedAt()`, `deactivatedAt()`, `maxVisits()`, and `expirationRedirectUrl()` methods.
  </Accordion>

  <Accordion title="v1.3.0 — Automatic Scheduler Registration">
    - Both Artisan commands (`short-url:sync-counters` and `short-url:aggregate-and-prune`) are now auto-registered by the service provider — no manual schedule entries required in your application.
    - The scheduler dynamically respects your Settings GUI toggles; disabling a feature in Settings stops the corresponding job from running.
    - Added real-time **Test connection** verify actions for the GA4 Measurement Protocol API and MaxMind database path in Settings.
    - Improved click-to-copy reliability in table rows with a built-in fallback for non-HTTPS browser environments.
  </Accordion>

  <Accordion title="v1.2.0 — Password Protection, Targeting Rules, and Aggregation">
    - **Password-Protected Links** — Session-based unlock flow with a styled prompt page; visitors only enter the password once per session.
    - **Redirect Warning Pages** — Interstitial safety screen before redirecting to external domains, with Continue and Go Back buttons.
    - **Smart Targeting** — Device-based, country/geo-based, and A/B weighted rotation rules per link.
    - **Rate Limiting** — Configurable per-IP redirect throttling returning `429 Too Many Requests` responses.
    - **Daily Stats Aggregation** — Nightly `short-url:aggregate-and-prune` command for scalable log management and configurable raw log pruning.
    - New **Performance & Security** settings tab for aggregation, pruning, and rate limiting configuration.
    - Full Polish (`pl`) locale translation support.
  </Accordion>
